## 🖼️ 图片管理

### 图片存储规范

| 类型 | 存储目录 | 引用路径 |
|------|----------|----------|
| 博客图片 | `public/images/blog/` | `/images/blog/xxx.jpg` |
| 产品主图 | `public/images/products/` | `/images/products/xxx.jpg` |
| 产品画廊 | `public/images/products/` | `/images/products/xxx-gallery1.jpg` |
| 运输证明 | `public/images/shipping/` | `/images/shipping/xxx.jpg` |
| 通用图片 | `public/images/general/` | `/images/general/xxx.jpg` |

### 图片命名规范

```
# 博客图片
blog-{slug}-{序号}.jpg
示例：blog-eu-ce-standard-1.jpg

# 产品图片
product-{slug}.jpg          # 主图
product-{slug}-gallery1.jpg # 画廊图1
product-{slug}-gallery2.jpg # 画廊图2

# 运输证明图片
shipping-{年份}-{月份}-{目的地}-{序号}.jpg
示例：shipping-2026-05-la-port-1.jpg

# 通用图片
general-{用途}.jpg
示例：general-banner.jpg, general-logo.png
```

### 图片格式要求

- **格式**: JPG 或 PNG（推荐 JPG，体积更小）
- **尺寸**: 
  - 博客封面：1200 × 800 px
  - 产品主图：800 × 800 px
  - 产品画廊：600 × 600 px
- **大小**: 单张图片不超过 500KB

## 🔄 更新流程

### 步骤一：修改内容

1. 编辑对应的 `.ts` 数据文件
2. 添加/修改/删除数据项

### 步骤二：上传图片

1. 将图片上传到 `public/images/` 对应目录
2. 在数据文件中引用正确的路径

### 步骤三：更新 Sitemap（新增页面时）

编辑 [scripts/generate-sitemap.js](file:///d:/b2bbaby/frontend/scripts/generate-sitemap.js)：

```javascript
// 如果新增博客文章，添加 slug 到 blogPosts 数组
const blogPosts = [
  'existing-post',
  'new-post-slug'  // 新增
];

// 如果新增产品，添加 slug 到 products 数组
const products = [
  'existing-product',
  'new-product-slug'  // 新增
];
```

### 步骤四：部署到服务器

```bash
# 1. 上传修改的文件到服务器
# 可以使用 FTP 或 Git

# 2. 进入前端目录
cd /www/wwwroot/b2bbaby/frontend

# 3. 重新构建
npm run build

# 4. 重启 PM2
pm2 restart b2bbaby-ssr
```

## 📋 常用操作示例

### 示例 1：新增博客文章

1. 在 `blog-data.ts` 中添加新对象
2. 在 `public/images/blog/` 上传封面图片
3. 在 `generate-sitemap.js` 的 `blogPosts` 数组中添加 slug
4. 执行构建和重启

### 示例 2：修改产品信息

1. 在 `product-data.ts` 中找到对应产品并修改
2. 如果需要更换图片，上传新图片到 `public/images/products/`
3. 更新数据文件中的图片路径
4. 执行构建和重启

### 示例 3：删除 FAQ

1. 在 `faq-data.ts` 中删除对应的对象
2. 执行构建和重启（不需要更新 sitemap）

## ⚠️ 注意事项

1. **路径格式**: 图片路径必须以 `/images/` 开头，不要使用相对路径
2. **Slug 唯一性**: 每个页面的 slug 必须唯一，不能重复
3. **缓存问题**: 更新后如果页面没变化，可能是浏览器缓存，按 Ctrl+Shift+R 强制刷新
4. **Sitemap 更新**: 新增页面后务必更新 sitemap，否则搜索引擎无法发现
5. **图片备份**: 定期备份 `public/images/` 目录
