# B2Bbaby 项目宝塔面板部署教程

## 📋 概述

本教程将指导您将 B2Bbaby 项目部署到服务器的宝塔面板中。项目采用 **Astro SSR + Directus CMS + Umami 统计** 架构：

| 组件 | 技术栈 | 部署方式 | 域名/端口 |
|------|--------|----------|-----------|
| 前端网站 | Astro 4 (SSR hybrid) + Tailwind CSS | 宝塔 PHP 站点 + Node.js PM2 + Nginx | b2bbaby.com → :4321 |
| Directus 后台 | Directus 11 + PostgreSQL 16 | Docker | admin.b2bbaby.com → :8055 |
| Umami 统计 | Umami + PostgreSQL 16 | Docker | umami.b2bbaby.com → :3001 |
| 邮件服务 | Nodemailer + SMTP | 前端 SSR 内置 | /api/send-email.json |

> 💡 **为什么用宝塔 PHP 站点部署前端？** 宝塔的 PHP 站点管理功能可以方便地管理 Nginx 配置、SSL 证书、网站目录，后期更新页面内容只需在服务器上重新 `npm run build`，无需重新构建 Docker 镜像，更新更快捷。

### 项目架构图

```
用户浏览器
    │
    ├── https://b2bbaby.com ──────► 宝塔 PHP 站点 (Nginx)
    │                                ├── 静态文件 → /www/wwwroot/b2bbaby/frontend/dist/client
    │                                ├── SSR 页面 → PM2 Node.js 进程 (:4321)
    │                                └── /api/send-email.json → PM2 → SMTP
    │
    ├── https://umami.b2bbaby.com ─► Nginx 反代 → Umami Docker (:3001)
    │
    └── https://admin.b2bbaby.com ─► Nginx 反代 → Directus Docker (:8055)
                                              └── PostgreSQL (:5432)
```

## 🔧 第一步：准备工作

### 1.1 服务器要求

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Ubuntu 20.04+ / CentOS 7+ | Ubuntu 22.04 LTS |
| 内存 | 2GB | 4GB+ |
| 硬盘 | 20GB | 40GB+ SSD |
| CPU | 1核 | 2核+ |
| 宝塔面板 | 7.0+ | 最新正式版 |

### 1.2 域名 DNS 配置

在域名服务商处添加以下 A 记录：

| 记录类型 | 主机记录 | 记录值 | TTL |
|----------|----------|--------|-----|
| A | @ | 服务器公网 IP | 600 |
| A | www | 服务器公网 IP | 600 |
| A | umami | 服务器公网 IP | 600 |
| A | admin | 服务器公网 IP | 600 |

> ⚠️ DNS 生效通常需要 10 分钟 ~ 48 小时，可用 `ping b2bbaby.com` 验证。

### 1.3 端口开放

确保以下端口已在服务器防火墙和云服务商安全组中开放：

| 端口 | 用途 | 是否必须对外 |
|------|------|-------------|
| 80 | HTTP | 是 |
| 443 | HTTPS | 是 |
| 4321 | Astro SSR（Nginx 反代） | 否（仅 127.0.0.1） |
| 8055 | Directus（Nginx 反代） | 否（仅 127.0.0.1） |
| 3001 | Umami（Nginx 反代） | 否（仅 127.0.0.1） |
| 5432 | PostgreSQL (Directus) | 否（仅 127.0.0.1） |
| 5433 | PostgreSQL (Umami) | 否（仅 127.0.0.1） |

> 🔒 安全建议：数据库和内部服务端口不要对外暴露，仅绑定 `127.0.0.1`。

### 1.4 宝塔面板安装必要软件

在宝塔面板 **软件商店** 中安装：

1. **Nginx**（必须，推荐 1.24+）
2. **Docker 管理器**（必须，用于可视化容器管理）
3. **PM2 管理器**（必须，用于管理 Node.js SSR 进程）

> 💡 PM2 管理器安装路径：宝塔面板 → 软件商店 → 搜索"PM2" → 安装。安装后会自动安装 Node.js 环境。

### 1.5 配置 Node.js 和 PM2 环境变量（重要！）

宝塔 PM2 管理器安装的 Node.js 和 PM2 在非标准路径下，命令行直接输入 `pm2` 会提示 `command not found`。**必须手动配置环境变量**：

**第一步：找到 Node.js 和 PM2 的实际安装路径**

```bash
# 查找 node 的安装位置
find /www/server -name "node" -type f 2>/dev/null

# 查找 pm2 的安装位置
find /www/server -name "pm2" -type f 2>/dev/null
```

通常输出类似：
```
/www/server/nvm/versions/node/v18.19.0/bin/node
/www/server/nvm/versions/node/v18.19.0/bin/pm2
```

**第二步：创建软链接到系统 PATH（推荐）**

```bash
# 将下面的 v18.19.0 替换为你实际查到的版本号
NODE_BIN=/www/server/nvm/versions/node/v18.19.0/bin

ln -sf $NODE_BIN/node /usr/local/bin/node
ln -sf $NODE_BIN/npm /usr/local/bin/npm
ln -sf $NODE_BIN/npx /usr/local/bin/npx
ln -sf $NODE_BIN/pm2 /usr/local/bin/pm2
ln -sf $NODE_BIN/pm2-runtime /usr/local/bin/pm2-runtime
```

**第三步：验证**

```bash
node -v     # 应输出 v18.x.x
npm -v      # 应输出 9.x.x 或 10.x.x
pm2 -v      # 应输出 5.x.x
```

> ⚠️ 如果 `find` 命令没有找到 Node.js，说明 PM2 管理器未正确安装，请先在宝塔软件商店中安装 PM2 管理器，或在 PM2 管理器设置中选择一个 Node 版本。

### 1.6 备选方案：手动安装 Node.js 和 PM2

如果宝塔 PM2 管理器安装失败，可手动安装：

```bash
# Ubuntu
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs

# 安装 PM2
npm install -g pm2

# 验证
node -v     # 需要 v18+
npm -v
pm2 -v
```

### 3.1 创建 .env 文件

进入项目根目录 `/www/wwwroot/b2bbaby/`，复制模板并修改：

```bash
cp .env-template .env
```

### 3.2 编辑 .env 文件

```env
# ==========================================
# 1. PostgreSQL 数据库密码（必须修改！）
# ==========================================
DB_PASSWORD=请替换为强密码_数字+字母+符号

# ==========================================
# 2. Directus CMS 配置
# ==========================================
# DIRECTUS_KEY：随机32字符（字母数字混合）
DIRECTUS_KEY=abcdefghijklmnopqrstuvwxyz123456
# DIRECTUS_SECRET：随机64字符
DIRECTUS_SECRET=abcdefghijklmnopqrstuvwxyzabcdefghijklmnopqrstuvwxyz12345678
# Directus后台管理员密码（登录 admin@b2bbaby.com 用）
ADMIN_PASSWORD=请替换为Directus后台密码

# ==========================================
# 3. Umami Analytics 统计配置
# ==========================================
UMAMI_DB_NAME=umami
UMAMI_DB_USER=umami
UMAMI_DB_PASSWORD=请替换为Umami数据库密码
# UMAMI_HASH_SALT：随机64字符
UMAMI_HASH_SALT=abcdefghijklmnopqrstuvwxyzabcdefghijklmnopqrstuvwxyz12345678

# ==========================================
# 4. SMTP 邮件配置（联系表单发邮件用）
# ==========================================
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
SMTP_TO=recipient@example.com

# ==========================================
# 5. 网站基础配置
# ==========================================
SITE_URL=https://b2bbaby.com
SITE_NAME=B2B Baby
```

### 3.3 生成随机密钥

可使用以下命令生成随机密钥：

```bash
# 生成32字符随机KEY
openssl rand -hex 16

# 生成64字符随机SECRET
openssl rand -hex 32

# 生成64字符HASH_SALT
openssl rand -hex 32
```

### 3.4 Gmail 应用专用密码

如果使用 Gmail SMTP，需要生成应用专用密码：

1. 访问 https://myaccount.google.com/apppasswords
2. 选择"邮件"和对应设备
3. 生成16位密码，填入 `SMTP_PASS`

> ⚠️ 不要使用 Gmail 账号主密码，必须使用应用专用密码。

## 🐳 第四步：启动 Docker 服务（Directus + Umami）

### 4.1 docker-compose.yml 说明

Docker 仅用于运行 Directus 和 Umami，前端不在 Docker 中运行：

| 服务 | 镜像 | 端口映射 | 说明 |
|------|------|----------|------|
| db | postgres:16-alpine | 5432:5432 | Directus 数据库 |
| directus | directus/directus:11 | 8055:8055 | CMS 后台管理 |
| umami-db | postgres:16-alpine | 5433:5432 | Umami 数据库 |
| umami | ghcr.io/umami-software/umami:postgresql-latest | 3001:3000 | 网站统计 |

### 4.2 修改 docker-compose.yml

需要从 `docker-compose.yml` 中移除 `frontend` 服务（前端改用宝塔 PHP 站点 + PM2 部署）。编辑 `/www/wwwroot/b2bbaby/docker-compose.yml`，删除以下部分：

```yaml
  # 删除这整个 frontend 服务块 ↓
  frontend:
    build: ./frontend
    ports:
      - "0.0.0.0:4321:4321"
    environment:
      HOST: "0.0.0.0"
      PORT: "4321"
      NODE_ENV: "production"
      SMTP_HOST: ${SMTP_HOST:-smtp.gmail.com}
      SMTP_PORT: ${SMTP_PORT:-587}
      SMTP_USER: ${SMTP_USER}
      SMTP_PASS: ${SMTP_PASS}
      SMTP_TO: ${SMTP_TO}
    restart: unless-stopped
    networks:
      - b2bbaby-network
```

修改后的 `docker-compose.yml` 应为：

```yaml
version: "3.8"

services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: directus
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/postgresql/data
    restart: unless-stopped
    networks:
      - b2bbaby-network
    ports:
      - "127.0.0.1:5432:5432"

  directus:
    image: directus/directus:11
    ports:
      - "127.0.0.1:8055:8055"
    environment:
      KEY: ${DIRECTUS_KEY}
      SECRET: ${DIRECTUS_SECRET}
      DB_CLIENT: "pg"
      DB_HOST: "db"
      DB_PORT: "5432"
      DB_DATABASE: "directus"
      DB_USER: "admin"
      DB_PASSWORD: ${DB_PASSWORD}
      ADMIN_EMAIL: "admin@b2bbaby.com"
      ADMIN_PASSWORD: ${ADMIN_PASSWORD}
      PUBLIC_URL: "https://b2bbaby.com"
      DIRECTUS_RUN_SCRIPT_ALLOW_LIST: "exec,spawn"
      REFRESH_TOKEN_COOKIE: "true"
      REFRESH_TOKEN_COOKIE_SECURE: "true"
      REFRESH_TOKEN_COOKIE_SAME_SITE: "Strict"
      PUBLIC_API_ENABLED: "true"
    volumes:
      - ./directus/uploads:/directus/uploads
      - ./directus/extensions:/directus/extensions
    depends_on:
      - db
    restart: unless-stopped
    networks:
      - b2bbaby-network
    security_opt:
      - seccomp:unconfined

  umami-db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: ${UMAMI_DB_NAME}
      POSTGRES_USER: ${UMAMI_DB_USER}
      POSTGRES_PASSWORD: ${UMAMI_DB_PASSWORD}
    volumes:
      - umami_db_data:/var/lib/postgresql/data
    restart: unless-stopped
    networks:
      - b2bbaby-network
    ports:
      - "127.0.0.1:5433:5432"

  umami:
    image: ghcr.io/umami-software/umami:postgresql-latest
    ports:
      - "127.0.0.1:3001:3000"
    environment:
      DATABASE_URL: postgresql://${UMAMI_DB_USER}:${UMAMI_DB_PASSWORD}@umami-db:5432/${UMAMI_DB_NAME}
      DATABASE_TYPE: postgresql
      HASH_SALT: ${UMAMI_HASH_SALT}
      GEOLOCATION_PATH: /data/GeoLite2-City.mmdb
    volumes:
      - ./umami-data:/data
    depends_on:
      - umami-db
    restart: unless-stopped
    networks:
      - b2bbaby-network

volumes:
  db_data:
  umami_db_data:

networks:
  b2bbaby-network:
    driver: bridge
```

> 🔒 注意：所有端口绑定已改为 `127.0.0.1`，不对外暴露，仅通过 Nginx 反向代理访问。

### 4.3 启动 Docker 服务

```bash
cd /www/wwwroot/b2bbaby

# 启动所有 Docker 服务（后台运行）
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看实时日志
docker-compose logs -f
```

### 4.4 验证 Docker 服务

```bash
# 检查 Directus 是否就绪
curl -s http://127.0.0.1:8055/server/health

# 检查 Umami 是否就绪
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:3001
```

> 💡 首次启动 Directus 需要初始化数据库，可能需要 30-60 秒。

## 🏗️ 第五步：构建前端并启动 SSR 服务

### 5.1 安装前端依赖

```bash
cd /www/wwwroot/b2bbaby/frontend

# 安装 npm 依赖
npm install
```

> ⚠️ 如果服务器网络较慢，可使用国内镜像：`npm install --registry=https://registry.npmmirror.com`

### 5.2 创建前端环境变量

前端 SSR 服务需要 SMTP 配置来发送邮件。创建 `/www/wwwroot/b2bbaby/frontend/.env`：

```env
# SMTP 邮件配置（联系表单用）
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
SMTP_TO=recipient@example.com

# Directus API 地址（SSR 构建时获取数据用）
DIRECTUS_URL=http://127.0.0.1:8055
```

### 5.3 构建前端

```bash
cd /www/wwwroot/b2bbaby/frontend

# 执行构建（含 sitemap 生成）
npm run build
```

构建完成后，产物分布：

```
frontend/
├── dist/
│   ├── client/     ← 静态文件（HTML/CSS/JS/图片），Nginx 直接服务
│   └── server/     ← SSR 服务端代码，PM2 运行
└── ...
```

### 5.4 使用 PM2 启动 SSR 服务

**方式一：宝塔 PM2 管理器（推荐）**

1. 宝塔面板 → **软件商店** → **PM2 管理器** → **设置**
2. 点击 **添加项目**
3. 填写：
   - **项目名称**: `b2bbaby-ssr`
   - **启动文件**: `/www/wwwroot/b2bbaby/frontend/dist/server/entry.mjs`
   - **运行目录**: `/www/wwwroot/b2bbaby/frontend`
   - **Node版本**: 18+
4. 点击 **提交**

**方式二：命令行 PM2**

```bash
pm2 start /www/wwwroot/b2bbaby/ecosystem.config.cjs
# 设置开机自启
pm2 save
pm2 startup
```

### 5.5 验证 SSR 服务

```bash
# 检查 PM2 进程状态
pm2 status

# 检查 SSR 服务是否响应
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:4321

# 查看 SSR 日志
pm2 logs b2bbaby-ssr
```

预期输出：HTTP 状态码 `200`，PM2 状态为 `online`。

### 5.6 PM2 常用命令

```bash
# 查看所有进程
pm2 status

# 查看日志
pm2 logs b2bbaby-ssr

# 重启 SSR 服务
pm2 restart b2bbaby-ssr

# 停止 SSR 服务
pm2 stop b2bbaby-ssr

# 删除 SSR 进程
pm2 delete b2bbaby-ssr

# 监控面板
pm2 monit
```

## 🌐 第六步：配置宝塔网站和 Nginx

### 6.1 创建主站（b2bbaby.com）

1. 宝塔面板 → **网站** → **添加站点**
2. 填写：
   - **域名**: `b2bbaby.com`（同时添加 `www.b2bbaby.com`）
   - **根目录**: `/www/wwwroot/b2bbaby/frontend/dist/client`
   - **PHP版本**: 纯静态
   - **数据库**: 不创建
3. 点击 **提交**

> 💡 使用宝塔 PHP 站点方式创建，方便后期在面板中直接管理 SSL、Nginx 配置、网站目录等。

### 6.2 配置主站 Nginx

> ⚠️ **重要**：不要替换整个配置文件！宝塔面板会自动生成包含服务器 IP 的 `include` 路径，直接替换会导致 Nginx 报错。请按以下步骤**增量修改**。

**操作方式**：进入 `b2bbaby.com` 网站设置 → **配置文件**，在宝塔自动生成的配置基础上，做以下修改：

#### 修改一：修改网站根目录

找到 `root` 行，修改为：

```nginx
root /www/wwwroot/b2bbaby/frontend/dist/client;
```

#### 修改二：添加性能优化配置

在 `#REWRITE-END` 之后、`location` 之前，添加：

```nginx
    # =====================================================
    # 性能优化
    # =====================================================
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    keepalive_requests 100;

    gzip on;
    gzip_min_length 256;
    gzip_buffers 4 16k;
    gzip_http_version 1.1;
    gzip_comp_level 6;
    gzip_types text/plain text/css application/javascript application/x-javascript application/xml application/json image/svg+xml text/javascript application/vnd.ms-fontobject application/x-font-ttf font/opentype;
    gzip_vary on;
    gzip_disable "MSIE [1-6]\.";

    client_max_body_size 100M;
```

#### 修改三：替换 location / 块

找到原有的 `location /` 块，替换为以下内容（静态优先，SSR 降级）：

```nginx
    # =====================================================
    # SSL 验证目录放行
    # =====================================================
    location ^~ /.well-known/acme-challenge/ {
        allow all;
        root /www/wwwroot/b2bbaby/frontend/dist/client;
        default_type text/plain;
        try_files $uri =404;
        access_log off;
    }

    # =====================================================
    # Astro SSR API - 联系表单邮件发送
    # =====================================================
    location = /api/send-email {
        proxy_pass http://127.0.0.1:4321;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }

    location = /api/send-email.json {
        proxy_pass http://127.0.0.1:4321/api/send-email.json;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
        proxy_send_timeout 60s;
    }

    # =====================================================
    # 静态资源缓存
    # =====================================================
    location ~ .*\.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot|webp|json|xml)$ {
        expires 30d;
        access_log off;
        add_header Cache-Control "public, immutable, max-age=2592000";
        add_header X-Content-Type-Options nosniff;
    }

    # =====================================================
    # Astro SSR 主页面（静态优先，SSR 降级）
    # =====================================================
    location / {
```

## ❓ 常见问题排查

### Q1: 前端页面刷新后 404

**原因**：Nginx 缺少 SSR 降级配置。

**解决方案**：确保 Nginx 配置中有以下内容：

```nginx
location / {
    try_files $uri $uri/ @astro_ssr;
}

location @astro_ssr {
    proxy_pass http://127.0.0.1:4321;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

> ⚠️ 本项目是 Astro SSR hybrid 模式，不是纯 SPA，不能用 `try_files $uri $uri/ /index.html`。

### Q2: 联系表单发送邮件失败

**排查步骤**：

1. 检查 `frontend/.env` 中 SMTP 配置是否正确
2. 检查 Gmail 是否启用了应用专用密码
3. 查看 SSR 服务日志：`pm2 logs b2bbaby-ssr`
4. 测试 SSR 服务是否响应：`curl http://127.0.0.1:4321/api/send-email.json`

### Q3: pm2: command not found

**原因**：宝塔 PM2 管理器安装的 PM2 不在系统默认 PATH 中。

**解决方案**：

```bash
# 1. 找到 PM2 的实际路径
find /www/server -name "pm2" -type f 2>/dev/null

# 2. 创建软链接（替换为你实际查到的路径）
ln -sf /www/server/nvm/versions/node/v18.19.0/bin/pm2 /usr/local/bin/pm2
ln -sf /www/server/nvm/versions/node/v18.19.0/bin/pm2-runtime /usr/local/bin/pm2-runtime

# 3. 验证
pm2 -v
```

如果 `find` 找不到 PM2，手动安装：

```bash
npm install -g pm2
```

### Q4: PM2 进程频繁重启

**排查步骤**：

1. 查看错误日志：`pm2 logs b2bbaby-ssr --err`
2. 检查端口是否被占用：`netstat -tlnp | grep 4321`
3. 检查 Node.js 版本：`node -v`（需要 v18+）
4. 手动启动测试：
   ```bash
   cd /www/wwwroot/b2bbaby/frontend
   node dist/server/entry.mjs
   ```

### Q5: Umami 无法访问

**排查步骤**：

1. 检查 Docker 服务状态：`docker-compose ps umami`
2. 检查端口：`netstat -tlnp | grep 3001`
3. 检查 Umami 日志：`docker-compose logs umami`
4. 检查 GeoLite2 数据文件是否存在：`ls -la umami-data/GeoLite2-City.mmdb`

### Q6: Directus 扩展未加载

**排查步骤**：

1. 检查 `directus/extensions/` 目录权限
2. 查看 Directus 日志：`docker-compose logs directus`
3. 确认 volume 映射正确：`docker-compose exec directus ls /directus/extensions`

### Q7: Docker 容器启动失败

**排查步骤**：

1. 查看详细日志：`docker-compose logs <服务名>`
2. 检查 `.env` 文件是否存在且格式正确
3. 检查端口是否被占用：`netstat -tlnp | grep <端口号>`
4. 检查磁盘空间：`df -h`

### Q8: SSL 证书申请失败

**排查步骤**：

1. 确认域名 DNS 已正确解析到服务器 IP
2. 确认 80 端口可从外网访问
3. 确认 Nginx 中 `.well-known` 目录放行配置正确
4. 查看宝塔面板 SSL 申请日志

### Q9: npm install 或 npm run build 失败

**排查步骤**：

1. 检查 Node.js 版本：`node -v`（需要 v18+）
2. 使用国内镜像：`npm install --registry=https://registry.npmmirror.com`
3. 清除缓存重试：`npm cache clean --force && rm -rf node_modules && npm install`
4. 检查内存是否充足（构建需要约 1GB 内存）
