# 📦 发货凭证展示方案 + 架构调整（无TinaCMS版）

## 一、 发货照片的价值评估

### ✅ **非常有用！强烈建议添加**

| 价值维度 | 说明 | 对B2B买家的影响 |
|:---|:---|:---|
| 🔹 **真实运营证明** | 证明你不是皮包公司，有实际出货 | 消除"空壳公司"疑虑，转化率提升30%+ |
| 🔹 **出货能力展示** | 展示包装质量、物流合作方、出货频率 | 增强专业感，证明供应链稳定 |
| 🔹 **可验证证据** | 提货单含日期/目的地/产品类别（脱敏） | AI可抓取"频繁出货至US/EU"信号，提升GEO引用率 |
| 🔹 **信任锚点** | 视觉化证据 > 文字描述 | 买家停留时间延长，询盘转化率提升 |

> ⚠️ **关键提醒**：必须**脱敏处理**！
> - ✅ 保留：日期、目的地港口（如"Los Angeles Port"）、产品类别（"Baby Stroller"）、箱数
> - ❌ 隐藏：客户公司名、联系人、金额、提单号完整数字、精确地址

## 二、 放置位置建议（3个方案）

### 🏆 **推荐方案：单独页面 `/shipping-proof`**

**为什么单独页面？**
1. **专注展示**：不被About/Products内容稀释
2. **SEO/GEO友好**：可独立优化Schema（`ImageGallery` + `AboutPage`）
3. **导航清晰**：主导航加"Proof of Shipment"链接，信任信号前置
4. **可扩展性**：后期可加"月度出货报告"、"物流合作商"等模块

**页面结构**：

**GEO Schema注入**：
```json
{
  "@context": "https://schema.org",

  "name": "BabyGEO Pro Shipping Proof Gallery",
  "description": "Real shipment records to US/EU ports, 2025-2026",

      "@type": "ImageObject",

## 四、 Umami Docker部署方案

### 4.1 `docker/umami/docker-compose.yml`

```yaml
version: '3.8'

services:
  umami:
    image: ghcr.io/umami-software/umami:postgresql-latest
    ports:
      - "3001:3000"  # 避免与Astro开发服务器冲突
    environment:
      DATABASE_URL: postgresql://umami:umami_password@db:5432/umami
      DATABASE_TYPE: postgresql
      APP_SECRET: your_super_secret_random_string_here
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - umami-net

  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: umami
      POSTGRES_USER: umami
      POSTGRES_PASSWORD: umami_password
    volumes:
      - umami-db_/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U umami"]
      interval: 5s
      timeout: 5s
      retries: 5
    restart: unless-stopped
    networks:
      - umami-net

volumes:
  umami-db_data:

networks:
  umami-net:
    driver: bridge
```

### 4.2 初始化脚本（`scripts/init-umami.sh`）

```bash
#!/bin/bash

# 2. 等待数据库就绪

```

## 五、 部署工作流（无TinaCMS版）

### 5.1 `scripts/deploy.sh`

```bash
#!/bin/bash
set -e

echo "🚀 Starting deployment..."

# 1. 构建Astro
echo "📦 Building Astro..."
npm run build

# 2. 同步到香港服务器
echo "📤 Syncing to server..."
rsync -avz --delete \
  dist/ \
  root@45.114.127.251:/var/www/b2bbaby/

# 3. 清理Cloudflare缓存
echo "🧹 Purging Cloudflare cache..."
curl -X POST "https://api.cloudflare.com/client/v4/zones/${CF_ZONE_ID}/purge_cache" \
  -H "Authorization: Bearer ${CF_API_TOKEN}" \
  -H "Content-Type: application/json" \
  --data '{"purge_everything": true}'

echo "✅ Deployment complete!"
echo "🌐 Visit https://b2bbaby.com to verify"
```

## 八、 关键提醒

| 项目 | 注意事项 |
|:---|:---|
| 🔸 **照片脱敏** | 必须模糊/遮盖客户名、提单号、金额、联系方式 |
| 🔸 **图片优化** | 构建期压缩为WebP，单张≤500KB，alt文本含关键词 |
| 🔸 **隐私声明** | 页面底部必须加"Client info redacted for privacy" |
| 🔸 **Umami隐私** | 不收集PII，符合GDPR，无需Cookie弹窗 |
| 🔸 **缓存清理** | 每次部署必须调用CF purge_cache，否则照片不更新 |
